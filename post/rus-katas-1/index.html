Описание задач Codewars на русском. Часть 1

September 09, 2020
Статьи

В данной серии статей будут публиковаться переводы задач по программированию на сайте Codewars. А также, при необходимости будут даны дополнительные объяснения, потому что иногда инструкции бывают непонятными.

Эти переводы рассчитаны на тех, у кого уровень знания английского языка (плюс начальный уровень программирования) не позволяет четко понять суть задачи.

Каты выбираются все подряд (по 5 на одну статью) от наиболее завершенной (Most Completed) и так далее. Т.е. сначала будут самые простые задачи, и, постепенно уровень будет усложняться.

Если вы на сайте Codewars впервые, то будет полезно прочитать другую мою статью о том, как им пользоваться.


Ката 1

Название: Multiply (ссылка)

Сложность: 8 kyu

Инструкции: Этот код не выполняется должным образом. Попробуйте выяснить почему.

// JavaScript
function multiply(a, b) {

}



Ката 2

Название: Vowel Count (ссылка)

Сложность: 7 kyu

Инструкции: Напишите фунцию, которая возвращает сумму гласных (a, e, i, o, u) в строке (string). Все вводимые значения в тестах будут состоять из прописных букв и / или пробелов

Рекомендации: Как вариант, можно воспользоваться циклом для проверки текущей буквы (символа), соответствует ли он гласной или нет. И если да, то к переменной vowelCount (количество гласных) прибавить единицу.

// JavaScript
function getCount(str) {
  var vowelsCount = 0;

  // напишите здесь свой волшебный код

  return vowelsCount;
}

getCount("cat"); // => 1, так как в слове 'cat' одна гласная буква 'a'.
getCount("codewars"); // => 3, так как в слове 'codewars' три гласные буквы 'o', 'e,' и 'a'



Ката 3

Название: Multiples of 3 or 5 (ссылка)

Сложность: 6 kyu

Инструкции: Функция должна возвращать сумму чисел кратных 3 и/или 5 от введенного значения. Если число кратно и 3 и 5, то это число к сумме прибавляется только один раз. Также, если число меньше нуля, вместо него возвращается 0. Сам аргумент функции при расчете не учитывается.

Рекомендации: Кратность в математике, это когда одно число делится на другое без остатка. В JavaScript это можно определить с помощью оператора остатка %. Например, 3 % 3 вернет остаток 0.

// JavaScript
function solution(number) {
  // ваш код
}

solution(10); // => 23, так как 3 + 5 + 6 + 9 = 23

// 10 кратно 5, но по условию задачи учитываются числа только до аргумента функции,
// в данном случае 10.

// JavaScript
function even_or_odd(number) {

even_or_odd(2); // => "Even"
even_or_odd(3); // => "Odd"
