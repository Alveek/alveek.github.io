Итак, вы установили Git с официального сайта, теперь необходимо открыть терминал и ввести команды, указав свое имя и электронную почту. Это делается для того, чтобы Git сохранял изменения под вашим именем.

    $ git config --global user.name "Elon Musk"
    $ git config --global user.email "elonmusk@tesla.com"

Ниже представлен список вопросов, которые могут возникнуть и ответы на них.



Я хочу создать Git репозиторий в моем проекте

> git init - эта команда создает в корне вашего проекта папку .git



Я передумал и хочу удалить репозиторий из моего проекта

> rm -rf .git - удаляет скрытую папку .git из вашего проекта.



Я добавил новый код. Как сохранить эти изменения в репозитории?

Сохранение в git происходит в несколько этапов:

  1. > git add . - (в конце именно точка) добавляет все файлы в вашем проекте на очередь для дальнейшего сохранения.

  2. > git commit -m 'Change button style' - подтверждение и сохранение изменений в репозиторий с соответствующим сообщением. (-m от “message”)



Я ввёл команду “git commit”, забыв добавить “-m ‘message’” и попал непонятно куда. Как выбраться оттуда?

На скриншоте ниже показано, что будет, если набрать > git commit и сразу нажать Enter. Добро пожаловать в Vim - текстовый редактор, которым до сих пор пользуются, потому что не знают, как оттуда выйти. Здесь несколько вариантов событий.

  1. Вы сразу выходите обратно в терминал. Для этого нажмите ESC, затем напечатайте :q и (это не смайлик, а двоеточие и буква ‘q’ (от “quit")) и затем нажмите ENTER.
  2. Вы пишите причину коммита, сохраняете и выходите. Для этого сначала напишите свое сообщение, нажмите ESC, напечатайте :wq и нажмите ENTER.



Я хочу поработать над новым функционалом программы, но пока не хочу это делать в основном проекте

Git позволяет создавать отдельные “ветки” (branch) вашего проекта, где вы можете экспериментировать. По умолчанию, когда других веток у вас еще нет - создается и используется “master” ветка.

> git branch - отображает все ветки, которые у вас есть.

> git branch new-feature - создает ветку под названием “new-feature”.

> git checkout new-feature - переход в выше созданную ветку. Все изменения в ней не коснутся master ветки.

> git checkout master - вернуться на главную (master) ветку репозитория.



Есть ли возможность проверить состояние репозитория? Может я что-то забыл сделать или есть какие-то рекомендации?

> git status - эта команда показывает на какой ветке вы находитесь, какие файлы отслеживаются, а какие нет. В примере ниже говорится о том, что файл в файле index.html есть изменения, но он не готов для коммита, а app.js вообще не отслеживается репозиторием.


После того, как мы ввели команды git add . и снова git status, мы видим, что теперь 2 файла готовы для коммита (т.е. надо ввести команду > git commit -m 'Причина коммита')



Я закончил работу на другой ветке, как теперь объединить её с master веткой?

Если вы еще не находитесь на главной ветке, перейдите туда, набрав команду:

> git checkout master

а затем:

> git merge new-feature - все изменения, которые вы сделали в ‘new-feature’ ветке появятся теперь в мастер ветке. Если ‘new-feature’ ветка вам больше не нужна, можно удалить её с помощью команды

> git branch -d new-feature



Я хочу хранить свои репозитории в интернете (облаке, сервисе), как резервные копии или для совместной работы

Для этой цели есть различные сервисы, но здесь мы рассмотрим GitHub.

Итак, вы зарегистрировались в GitHub, теперь создайте новый репозиторий (+ new repository).

  • введите название репозитория
  • решите будет ли он публичным или приватным.
  • не ставьте галочку, где написано “Initialize this repository with a README” (для того, чтобы появилось окно с инструкциями)
  • нажмите Create repository.

Появится следующее окно с инструкциями:

Здесь нас интересуют эти команды:

Можно их скопировать и вставить в командную строку с вашим проектом. Если вы делаете это в первый раз, скорее всего вас попросят ввести данные от GitHub аккаунта.

Проект появится в GitHub:

Теперь работа с Git будет выглядеть так:

  1. > git add . - добавить изменённые файлы в очередь для дальнейшего сохранения в репозитории.
  2. > git commit -m 'Причина изменений'
  3. > git push - отправить измененные файлы в GitHub.



В GitHub хранится более поздняя версия репозитория, чем у меня локально, в компьютере. Как мне загрузить эту последнюю версию?

Когда вы работаете с кем-то над одним проектом (или работаете один, но переодически на разных компьютерах), вам необходимо будет загружать из вашего общего репозитория последние изменения, сделанные вашим напарником (или вами с другого источника).

> git pull - загрузить последнюю версию репозитория из GitHub и обновить локальный.



Я не хочу, чтобы Git индексировал конкретную папку или файл

Для этого нужно создать файл .gitignore и вписать название папки или файла, которые git должен игнорировать.
